# L1 BASIC

インタプリタの解釈ルーチン自体が中間言語を用いて作られている（二重インタプリタ方式）。速度はかなり遅くなる。

中間言語はテキサス版Tiny BASIC（DDJ 1976.2） に類似であり、テキサス版（8080版）を6800に移植したものと思われる。

解析には8080用の東京版Tiny BASIC（テキサス版を元に作成）の資料が参考になる。

- [マイクロコンピュータプログラミング入門 : Tiny BASIC インタプリタ (コンピュータサイエンス大学講座) - 国立国会図書館デジタルコレクション](https://dl.ndl.go.jp/pid/12631493/1/31)

$D600- ($D5FE?）から中間言語で書かれた解釈実行ルーチンが存在する。
（BASICが中間言語になっているのではなく、BASICインタプリタ自体が中間言語で書かれている）

## 中間言語インタプリタ

$D594から中間言語のインタプリタ。

命令はMCALL,ICALL,JUMP,TESTの4種。全て2バイト命令。

第1バイトの上位2bitで命令種別を判断する。

- $00-$3F: ICALL a
- $40-$7F: JUMP a
- $80-$BF: TEST a,s
- $C0-$FF: MCALL a

実効アドレスは下位14bitと$C000をORしたアドレスとなる。

L1 BASIC ROMは$D000-$DFFFにあるため、使用される命令範囲は下記の通りとなる。

- $10-$1F: ICALL a
- $50-$5F: JUMP a
- $90-$9F: TEST a,s
- $D0-$DF: MCALL a


## 中間言語

$00DC-$00DD: 中間言語のプログラムカウンタ
$00DE-$00DF: BASICソースコードの読み出しポインタ

### $00-$3F（%00000000-%00111111）：ICALL a

$D5B5- で処理する。

$C000とORを取り、そのアドレスの中間言語サブルーチンを実行

### $40-$7F（%01000000-%01111111）: JUMP a

$D5BF- で処理する。

$C000とORを取り、そのアドレスへ分岐する（中間コードのまま分岐）。


### $80-$BF（%10000000-%10111111）: TEST a,s

2バイト命令。$D5CF- で処理する。

$00DEからBASICソースコードを読み、中間言語に続く文字列と比較する。
文字列は最後のバイトのb7が1になっている。

一致してなければ、 $C000とORを取ったアドレスへ分岐する（中間コードのまま分岐）。
一致していた場合は、 次の中間言語命令を実行する。

一致処理にSPを使っている（処理中はSEIを行っている）ので、タイマーが正しく動作しないはず。TIME関数はないので問題にならないのかも。


### $C0-$FF（%11000000-%11111111）: MCALL a

2バイト命令。$D5A0- で処理する。
命令コードをアドレスと見做して、その番地の機械語サブルーチンを実行する。

サブルーチンから戻ってきたときに、C=1であれば、次の命令をスキップする（2バイトスキップ）。
